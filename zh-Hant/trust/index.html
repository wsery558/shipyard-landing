信任中心

Shipyard 追求的是「看得懂、查得到、交得出去」。這頁用最務實的方式說清楚：我們怎麼處理安全、隱私與負責任揭露（不做過度承諾）。

加入候補名單 Evidence Pack

安全概覽

  • 可輸出證據：活動紀錄與產物能夠整理成即可檢視的包。
  • 完整性支援：checksums 協助在交付時發現可能的改動。
  • 少講保證、多給證據：以可驗證輸出取代空泛承諾。

隱私

這是產品網站。若你送出候補表單，你的回覆會由表單服務商保存。 另外我們會收集輕量使用事件（頁面瀏覽、點擊）來改進網站與產品。

  • 會收集：頁面路徑、時間戳、語言、以及可能存在的 referrer/UTM。
  • 不鼓勵提供：密碼、API key、私有原始碼等敏感資料。
  • 資料使用：不販售個資。

若你對資料處理有疑問，請看下方聯絡方式。

支援

最快的方式是透過 email。若你已購買並需要 Evidence Pack 的流程，請到候補頁貼上你的收據 email，我們會用 email 回覆你。

Evidence Pack 候補入口

負責任揭露

我們歡迎善意的安全回報。請附上重現步驟、影響描述，以及可安全分享的證據。 我們會儘快回覆並協調修正，但不保證時效。

  • 回報方式：請參考 security.txt 與 SECURITY.md。
  • 範圍：本網站、公開 demo 與 Shipyard 相關 repo。
  • 善意保護：避免傷害與擴散的測試，我們視為善意回報。

更新紀錄

產品與 repo 的變更以 GitHub commits / releases 為準；若有公開 changelog，會在這裡補連結。

聯絡

安全與隱私問題請使用 security.txt 中的聯絡方式；一般問題請用候補表單，我們會用 email 回覆。
